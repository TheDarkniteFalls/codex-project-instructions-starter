# Project Instructions

Use this file to tell coding agents how to work in this repository.

## Scope

- Start from the current repo state.
- Inspect relevant files before editing.
- Keep changes focused on the requested task.
- Do not rewrite unrelated code, docs, history, or generated files.

## Public And Private Boundaries

- Use synthetic fixtures for public examples.
- Do not add credentials, personal data, connector exports, raw logs, private
  notes, or local machine paths.
- Ask before publishing, pushing, or making a private repo public.

## Checks

- Run the smallest relevant check before committing.
- Prefer one named project health command when the repo has one.
- Report any check that could not be run.

## Handoff

End substantial work with:

- what changed
- which files were touched
- which checks passed
- what remains risky or unproven
- the next best action
